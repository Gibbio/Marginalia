# Security Policy

## Supported Status

Marginalia is in early bootstrap and does not yet provide hardened production
security guarantees. Security fixes should still be treated as high priority.

## Reporting

Please do not open public issues for suspected vulnerabilities.

Instead, report them privately to the repository owner with:

- a concise description of the issue
- affected component or file path
- reproduction steps or proof of concept
- impact assessment
- suggested mitigation if known

If a formal security contact address is established later, this file should be
updated immediately.

## Scope Notes

Current likely security-sensitive areas:

- future local model and provider integrations
- document ingestion and file path handling
- local note persistence
- any later desktop shell or local API
